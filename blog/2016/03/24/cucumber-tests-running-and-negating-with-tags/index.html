Cucumber Tests Running and Negating With Tags

Mar 24th, 2016 12:15 am

Instead of running the entire test suite Cucumber gives us the option to run only a particular set of tests using Tags.
Example: cucumber --tags @unit

If more than one tag should be executed then a comma separated list of values will do the trick. Example: cucumber --tags @unit,@integration
Here the comma separated annotations is considered as Logical OR condition

But I was tasked to NOT to run a set of tests which have the specified tag (Because those tests should run separately).
Cucumber is giving an option to do that as well. We can just use cucumber --tags ~@donotrun

Wait what if I don’t want to run tests from more than one tag. If think like: cucumber --tags ~donotrun,~neverrun then you are wrong. As mentioned some lines before the comma between the tags represents logical OR.

So it becomes cucumber --tags (~donotrun || ~neverrun) which practically runs all the tests without negating anything.

Solution:
If the comma is logical OR then what we need is logical And. That is achieved by calling the –tags option again. cucumber --tags ~donotrun --tags ~neverrun
